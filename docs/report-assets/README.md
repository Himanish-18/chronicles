# Report Assets

This directory contains screenshots and images referenced in `PROJECT_REPORT.md`.

## Required Screenshots

To complete the report, capture the following screenshots and save them in this directory:

### Application Screenshots
- `screenshot-landing.png` — Landing page with hero section, featured stories, and trending blogs
- `screenshot-login.png` — Login page with email/password form and social login buttons (Google, GitHub)
- `screenshot-dashboard.png` — User dashboard with sidebar navigation and overview statistics
- `screenshot-editor.png` — Blog creation page with title, content, category selector, and cover image upload
- `screenshot-blog-detail.png` — Full blog post page with cover image, author card, and comment section
- `screenshot-categories.png` — Categories page showing all available categories
- `screenshot-themes.png` — Side-by-side comparison of dark and light themes
- `screenshot-bookmarks.png` — Dashboard bookmarks page showing saved blogs

### Deployment Screenshots
- `screenshot-docker-ps.png` — Terminal output of `docker compose ps` showing all services running
- `screenshot-github-actions.png` — Successful GitHub Actions workflow run from the Actions tab
- `screenshot-ec2.png` — AWS EC2 instance dashboard showing the running instance

## Capture Instructions

1. Navigate to the deployed application at `http://<EC2_PUBLIC_IP>:3000`
2. Use your browser's screenshot tool or a screenshot application
3. Save each image in PNG format in this directory
4. The report references these files by their exact filename
